CLO-2 Why Sorting is the Critical First Step

▶

The Role of Sorting

The greedy algorithm's correctness hinges on processing activities in non-decreasing order of finish time. Without sorting, the greedy choice at each step may not be globally optimal.

✓ Sort by Finish Time → Always Optimal

Guarantees the Greedy-Choice Property: the locally optimal pick leaves maximum room for future activities.

⚠ Any Other Sort → Sub-optimal or Wrong

Sorting by start time, duration, or random order can miss the optimal solution — demonstrated below.

CLO-3 Greedy Paradigm — How It Works

▶

✓ Paradigm: GREEDY

At each step, pick the unscheduled activity with the earliest finish time that doesn't conflict. This local optimum is provably globally optimal.

Two Properties That Make Greedy Work

1. Greedy-Choice Property: A globally optimal solution can always include the greedy first choice (earliest finish). Proven by the exchange argument below.

2. Optimal Substructure: After selecting the first activity, the remaining subproblem (compatible activities) has the same structure — apply greedy again.

CLO-3 The Exchange / Swapping Argument

▶

Formal Proof Sketch

Let A* be optimal with first activity a₁'. Let a₁ be greedy (earliest finish). Since f(a₁) ≤ f(a₁'), replacing a₁' with a₁ in A* still yields a valid solution of the same size → greedy is never worse.

CLO-3 Counterexample — Sort by Start Fails

▶

Why Other Paradigms / Sort Orders Fail

A[1,10], B[2,3], C[4,5]: Sort by start → greedy picks A first (starts earliest), blocks B and C → only 1 selected. Sort by finish → picks B, C → 2 selected (optimal). The greedy choice property only holds when sorted by finish time.

Why Not Dynamic Programming?

DP also finds the optimum but costs O(n²) for naive weighted interval scheduling. Since all weights are equal, greedy dominates: simpler code, O(n log n) time.

What is the Activity Selection Problem?

▶

Problem Statement

Given n activities each with start time sᵢ and finish time fᵢ, select the maximum number of non-overlapping activities.

Compatibility

Activities i and j are compatible if fᵢ ≤ sⱼ or fⱼ ≤ sᵢ.

Algorithm Steps

1. Sort by finish time — O(n log n)
2. Select first activity
3. For each remaining activity: if it starts at or after the last selected activity's finish time, select it
4. Return selected set

Reference

Cormen et al., Introduction to Algorithms, 4th ed., Ch. 16.1 — "An activity-selection problem"

Key Concepts & Common Mistakes

▶

✓ Sort by Finish Time Always

The single most important rule. Everything else follows.

⚠ Mistake: Sort by Start Time

Example: A[1,10], B[2,3], C[4,5]. Sort by start → picks A → blocks everything → 1 selected. Sort by finish → picks B, C → 2 selected.

⚠ Mistake: Sort by Duration (Shortest First)

Counter: B[0,3], A[2,4], C[3,6]. Durations: A=2, B=3, C=3. Picks A (shortest), but A conflicts with both B and C → 1. Greedy-by-finish picks B then C → 2.

💡 Ties in Finish Time

If two activities share the same finish time, both choices lead to equally valid solutions. Tiebreak by start time for consistency.
